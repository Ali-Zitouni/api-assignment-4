What Is The Project ?

The project is about: Weather Forecasting. Weather forecasting is the application of Science and Technology to predict the conditions of the Atmosphere for a given Location and Time. Weather impacts nearly every areas of our lives – Weather conditions such as fog, rain, and snow impact commutes to work and travelling around the globe So, I decided to manage the services of Weather Forecasting by using APIs. So, We are all aware of the Weather conditions in our area.

How It Works ?

With the Weather API everyone can retrieve current weather observations from over 47,000 live weather stations, historical weather data for the past 10 years from our archive of more than 120,000 stations, and highly localized weather forecasts for any point on the globe using the world's most trusted weather models!

Methods To Access Weather Data :

  • Latitude/longitude
  • City name
  • City ID
  • Weather station ID
  • Airport ICAO code
  • Postal (zip) code for any country in the world

Client Request :

The first method handles the get request (i.e., visiting the page with the named URL) for the server.
Here we use the unirest object for preparing and sending a get request to the API. It has a complete endpoint address, headers for RapidAPI and all the required params: appid, longitude and latitude, units and mode for returning response already in HTML.
The next step is to create a callback function unirest.get().end(), that creates a dynamic HTML response via writeHead() and write() methods.
Finally, it logs to the console with the phrase “Colosseum, I am coming!”.
To run our server, we used the app.listen() method (note that 8081 – port number for the localhost URL).
